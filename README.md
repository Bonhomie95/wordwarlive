# WordWar Live

Real-time 1v1 word game. Two players, same hidden word, 90 seconds to solve. Wordle meets head-to-head competition.

## Repo layout

```
wordwar/
├── server/    Node + Express + TypeScript + Socket.io + Postgres + Redis
└── mobile/    Expo SDK 54 + React Native + TypeScript
```

The two folders are independent projects. Shared protocol types are mirrored in `server/src/types/index.ts` and `mobile/src/types/index.ts` — keep them in sync when you change the wire format.

## Quick start

```bash
# 1. Server
cd server
cp .env.example .env       # fill in GROQ_API_KEY, OAuth client IDs
docker compose up -d       # spins up Postgres + Redis
npm install
npm run migrate            # runs db/migrations/*.sql
npm run seed               # seeds the word bank
npm run dev                # starts on :4000

# 2. Mobile (in a second terminal)
cd mobile
cp .env.example .env       # set EXPO_PUBLIC_API_URL to http://<your-lan-ip>:4000
npm install
npm start                  # opens Expo dev server
```

## Auth

Three methods, all unified into a single `User` record:

- **Anonymous** — username + device-generated UUID, instant play
- **Email + password** — bcrypt hash, JWT
- **OAuth** — Google Sign-In, Apple Sign-In (id_token verified server-side)

All paths produce the same JWT for use over HTTP and as the Socket.io `auth.token`.

## AI (Groq)

Two integrations, both via `llama-3.3-70b-versatile`:

1. **Daily word selection** — picks the daily featured word from the curated bank with a difficulty score and a one-line "theme" hint surfaced post-game.
2. **Bot opponents** — when matchmaking can't find a human within 20s, a bot joins. Bot guesses are generated by Groq with a difficulty parameter scaled to the user's rank tier. Bots are clearly labeled as bots in the UI.

## Anti-cheat principles

- The target word **never** leaves the server until `GAME_OVER`.
- Every guess is validated and scored server-side; clients only receive the per-tile result.
- Guesses are rate-limited to 1 per 2 seconds per player per match.
- All rank changes are computed server-side.
